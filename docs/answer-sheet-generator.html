问题：HTML → PDF 的一致性陷阱

CSS mm 单位解释分歧、SVG mm 不被 Chromium 解析、repeating-gradient 不被支持——三层翻译全部出错。

📐

CSS mm 单位解释分歧

CSS 的 1mm = 3.78 CSS px，但 ReportLab 认为 1mm = 72/25.4 pt，WeasyPrint 又有自己的换算。同一个"7mm"，三个引擎算出三个不同的物理值。

🎨

CSS Gradient 不被支持

repeating-linear-gradient 是 CSS3 特性，大部分 PDF 转换器（ReportLab、wkhtmltopdf 早期版）根本不支持，Guide Lines 直接消失或渲染错误。

⚙️

SVG mm 单位不生效

换成 SVG line + stroke-width="0.2mm"，Chromium PDF 引擎不认 SVG 的 mm 单位，Guide Lines 间距直接偏到 9.54mm——比理论值大 36%。

核心方案

不用 CSS Gradient，用 SVG line。不用第三方转换器，用 Chromium 本身。
